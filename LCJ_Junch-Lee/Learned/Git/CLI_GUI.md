## CLI / GUI

- GUI (Graphic User Interface)

: Graphic을 통해 유저와 컴퓨터가 상호작용

ex. 사용자가 원하는 것을 직접 터치 등을 통해 입력하는 방식


- CLI (Command Line Interface)

: 명령어를 줄 단위로 작성하여 상호작용 

ex. 내가 원하는 바를 컴퓨터에게 직접 입력 

- 효율성 : 키보드만으로 모든 작업을 빠르고 가볍게 수행할 수 있음 
- 정밀한 제어 : 그래픽 화면에 보이지 않는 숨겨진 설정까지 정밀하게 제어할 수 있음
- 강력한 자동화 : 단순하고 반복적인 작업을 명령어 한 줄로 자동화

****

### CLI 내에서의 여러 명령어

- cf. Tab을 눌러 자동완성 가능

&nbsp;

- touch : 파일 생성 

- mkdir (make directory) : 폴더 생성

- ls : 현재 경로 내의 모든 파일/폴더를 출력

&nbsp;

- cd (change directory) : 경로 변경 
    - . : 현재 파일 경로
    - .. : 상위 경로로 이동

- start : 파일 실행

- rm(remove) : 파일 지우기
  - -r 옵션 : recursive (재귀적) - 폴더를 지움 

&nbsp;

- pwd : 현재 작업 중인 디렉토리의 절대 경로를 출력


## CLI에서 가장 중요한 것 ? 

> :
> 
> 내가 **어디 있는지(경로)** 알아야 한다.
> 
> 경로란, 내가 있는 곳을 표현하는 방법

---
### 내가 어디에 있는가?

컴퓨터는 수많은 방으로 이루어진 거대한 건물과 같음 

: 경로(Path)는 특정 방(디렉토리)이나 물건(파일)을 찾아가는 주소 역할 ! 

'내가 어디에서 작업하고 있는가'를 이해하는 것이 CLI의 첫 걸음 ! 

- 루트 디렉토리 : 일종의 정문이자 시작점 

- ~ (홈 디렉토리) : 나의 '집', 터미널 시작 시 지정되는 기본 공간 

&nbsp;

### 절대 경로(Absolute Path) 

    : 정문(루트,/) 부터 목적지까지의 전체 주소 

    ex. /c/Users/SSAFY

### 상대 경로(Relative Path)

    : '현재 내 위치'를 기준으로 한 주소 

    ex. '저 방으로 가' (Desktop/) 또는 '이전 방으로' (...)와 같음
