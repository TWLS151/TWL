일종의 변화 관리 추적 

- 최종본을 통해 변경 사항들을 확인할 수 있다. 

![GIT 예시](./002.jpg)

- PJT 소스코드를 따로 관리해줄 수 있다.

## 1. 중앙 vs 분산

- 중앙 집중식

    : 버전은 중앙 서버에 저장, 파일을 가져와 다시 중앙에 업로드


- 분산식

    : 버전을 여러 개의 복제된 저장소에 저장 및 관리

  - 중앙 서버 의존에서 자유로움
  - 백업과 복구가 용이
  - 인터넷 없이 작업 가능 

&nbsp;


## 2. GIT의 역할
- 코드의 버전(히스토리)을 관리
    - 개발되어온 과정 파악

- 이전 버전과의 변경 사항 비교 (변경 이력 기록)

- 3영역 : Working Directory, Staging Area, Repository

- Working Directory

  : 실제 작업 중인 파일들이 위치하는 영역
  - 우리가 GIT 환경에서 직접 보게 되는 화면

- Staging Area (**`git status`**)
  
  : WorkDir에서 변경된 파일 중 다음 버전에 포함시킬 파일들을 선택적으로 추가 

  - 레포지토리에 저장되기 위해 반드시 거쳐야 함
  - Staging Area에 올렸다고 하더라도, 반드시 버전으로 함께 저장되어야 하는 것은 아님
  - **빨간색 처리 : Working dir에 있음을 의미**

- Repository(저장소, git log)
  
  : 버전 이력과 파일들이 영구적으로 저장되는 영역
  - 모든 버전과 변경 이력이 기록됨
  - 버전 = commit, 사진 찍듯이 기록한다 하여 'snapshot'이라고도 함


## 3. GIT 동작

- git init : 초기화, 프로젝트 폴더 임을 표시

  - (master) : 잘 설정됨


- git status :

  - 변경 사항을 수시로 저장
  - 파일을 추가하고 git add <file> 해주지 않을 경우, 'untracked' (U) 라고 출력

- git add : git에 파일을 추가, Added (A) 

  - 현재 Work dir의 모든 파일을 add -> '.(현재 경로)' 사용

- git commit -m "first commit"

    - 누구인지 밝혀라 
    - git config -- global user.name/email "이름/이메일"
      - --global : 컴퓨터 전체 지정
      - --local : 현재 파일에서만 지정 

- git log : 기록 확인

---
### 로컬 (local)
-  현재 사용자가 직접 접속하고 있는 기기 또는 시스템
- ex. 개인 컴퓨터, 노트북, 태블릿 등

---
### 주의사항

- Git 로컬 저장소 내에 또 다른 Git 로컬 저장소를 만들지 말 것 
  
  - Git 저장소 간의 상/하위 구조를 만들지 말 것

- 만약 잘못 만들었다면? 
    - 하위 경로의 숨겨진 `.git` 폴더를 삭제하면 됨 
    - `.(파일명)` 파일 : 숨겨진 파일 (환경설정 파일인 경우가 대부분)
