### 2. 튜플 (immutable)

- 불변성 제한 등으로, 우리가 자주 사용하지 X 

- 불변성을 가지는 이유?

  - 주로 **파이썬의 내부 동작과 안전한 데이터 전달**에 사용됨
  - 다중 할당, 값 교환, 함수 다중 반환값 등
  - cf) 언패킹 (추후)

### 3. range()

: 연속된 정수 시퀀스를 생성하는 변경 불가능한 (immutable) 자료형

- range(stop)
  - 매개 변수가 하나면 stop으로 인식
  - start = 0, step = 1이 기본값
  - stop 값은 포함되지 않음

```python

print(my_range_1) = #range(0,5)

# range 객체로만 존재
# 아직 평가되지 않음. 

# 지연 평가(lazy evaluation) 
# : 필요할 때만 값을 계산해주는 방식
```

#### range 형 변환
- list로 형 변환 시 값을 확인할 수 있음

#### range step 규칙

- step이 양수, 음수일 때 stop / start 값이 커야함

## **4. dict**

- **key-value** 쌍으로 이루어진 
- **순서**와 중복이 **없는** (비시퀀스, 인덱싱 X) 
- **변경 가능**한 자료형 

---

### 딕셔너리 표현

- 값 1개는 Key - Value 쌍
- Key는 중복 불가 (Value는 가능)

### Key, Value의 규칙

Key : 고유성, 변경 불가능한 자료형 
Value : 어떤 자료형도 가능 

### 딕셔너리 속성

- 값 호출 : Key를 사용해 Value를 호출할 수 있음 

`print(my_dict['name']) #홍길동`

- 값 추가(새로운 Key), 변경(기존 key)
- 삭제도 가능하나, 추가적인 함수가 필요
- dict value에 또 다른 dict가 가능

## 5. set (mutable)

: 순서와 중복이 없는 변경 가능한 자료형

- dict와 혼동을 피하기 위해, 비어있는 세트는 `set()`을 사용해야 함 

### set의 집합 연산

- 합집합 : 파이프라인 연산자 (`|`)를 활용
- 차집합(`-`), 교집합(`and`)

## 8. 형 변환 (Type Conversion)

### 암시적 형변환
- 파이썬이 연산 중 자동으로 데이터 타입을 변환
- 데이터 손실 방지

```python
print(3 + 5.0) # 8.0

# 파이썬이 정수 + 실수 덧셈에서 정수를 실수로 변경

print(True + 3) #4

# True -> int

print(True + False) #1

# boolean -> int (1 + 0)

```

### 명시적 형변환

- 개발자가 변환타입을 직접 함수로 지정
- str -> int 
  - 형식에 맞는 경우만 가능

- int -> str : 모두 가능 


## 9. 연산자

### 복합 연산자

## 10. 비교 연산자

- ==, != : 값이 같은지 (동등성)
- is, is not : 객체 자체가 같은지 (식별성)
  - 메모리 주소까지 비교

- 결론 : `==`를 사용하는 것이 기본

- why?
  - 우리가 두 값을 비교할 때, 값이 같은지에 주목해야 하지 어느 방에 들어있냐를 보는 것이 아님! 

- ### is는 언제 활용하나?

- 싱글턴 객체를 비교할 때 사용
  - 프로그램 전체에 오직 1개만 존재하도록 만들어진 객체 (1개의 방만 가질 때) 
  - None, True, False 에만 

## 11. 논리 연산자

- 비교 연산자와 함께 사용

### 단축 평가

- 필요한 계산 이후, 이미 결론이 났다면 추후의 계산은 진행하지 않음 

- 거짓 취급값 : None, False, 0, 빈 리스트 등 ('비어있다'고 판단되는 값)
- 참 취급값 : 그 외 모두 

## 12. 멤버십 연산자

- 특정 값이 시퀀스, 다른 컬렉션 내에 포함되어 있는지 확인하는 연산자 
- in, not in 

## 13. 시퀀스형 연산자

- `+` : 결합 연산자
- `*` : 반복 연산자
