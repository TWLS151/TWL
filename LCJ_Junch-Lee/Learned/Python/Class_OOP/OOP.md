## 절차 지향 프로그래밍

: Procedural Programming

: 함수와 로직 중심의 작성, 데이터를 순차적으로 처리

### 특징

- 변수와 함수를 별개로 사용

- 로직에 맞게 함수가 단계 별로 실행이 되는 것 (행위)가 중심

- 변수는 중요치 X 

- 순차적인 명령어 실행이 중요

  
### 한계

- 복잡성 증가에 따른 관리의 어려움

    - 프로그램 규모가 커질 수록 데이터(전역 변수), 함수의 관리가 어려움

- 문제 발생 시 유지/보수가 어려움


## 객체 지향 프로그래밍 (OOP)

: Object-Orientied Programing

: 순서 보다, 데이터가 중심(주체)이 되어 데이터가 행동을 하는 개념

- 행동하는 주체가 바로 '객체' 

- 클래스는 설계도, 인스턴스는 실제 물건


ex. 자동차 설계도 - 클래스, 조립된 자동차 - 인스턴스


#### 객체 지향 사고 예시

사람 (객체) 안에 name, age와 이와 관련된 기능(메서드) 포함

```python

class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    
    def introduce(self):
        print(f"안녕하세요, 저의 이름은 {name}이고, 나이는 {age}입니다.")

alice = Person("Alice", 25)

alice.introduce()
```


### 특징

- 데이터와 처리하는 함수(메서드)를 하나의 단위(객체)로 묶어서 조직적으로 관리하자 (데이터와 메서드의 결합)

- 처리 순서 보다는, **객체 간 상호작용**이 중요

- 데이터의 능동화 : 데이터의 역할이 더욱 커짐 (<-> 절차 지향 : 함수의 재료에 국한)

- 코드의 구조화와 재사용성을 높임 

- 실제 세계의 모델링 방식과 더 유사한 프로그래밍
