## 클래스 상속

: 부모 클래스의 속성과 메서드를 자식 클래스가 물려받는 것

1. 코드 재사용
- 부모 클래스의 속성, 메서드를 그대로 가져올 수 있음
- 메서드 정의 수를 줄일 수 있다

2. 계층 구조
- 하위 클래스에 더욱 세부적인 메서드 정의 가능


3. 유지 보수의 용이성
- 일관성 유지, 수정 범위의 최소화

ex. 게임의 캐릭터 클래스 - 전사, 마법사 클래스 간 상속 관계 구현

#### 자식 클래스 정의 : 반드시 부모 클래스 이름을 함께 선언

## 다중 상속

- 둘 이상의 상위 클래스로부터 여러 행동과 특징을 상속받을 수 있음

- 중복된 속성/메서드가 있는 경우, 상속받은 순서에 따라 결정


### 다이아몬드 문제

- B,C가 A로부터 상속되고, D가 B,C를 모두 상속받을 때, B와 C 모두에 정의된 속성/메서드 중 어떤 것을 상속받을 것인가?에 대한 문제

### MRO (Method Resolution Order)

: 파이썬이 메서드를 찾는 순서에 대한 규칙

1. 자식 클래스가 우선

2. 없을 시, 왼쪽 부모(먼저 상속받은 클래스) 우선

3. 중복 방문은 방지 (단 한 번만 탐색)

ex. 위의 예시 : D -> B -> C -> A

- 클래스 구조 설계 시 이를 개발자가 고려해야 함 ! 


## super() 메서드

- 올바른 순서(MRO)에 따라 부모 클래스의 속성/메서드에 접근할 수 있게 하는 메서드

- 부모의 메서드를 그대로 가져온 뒤, 자식 메서드에서의 추가 사항만을 정의해줄 수 있음


#### 다중 상속에서의 `super()` 사용 예시

(in 06_multiple_)

- 다중 상속에서 super()는 '부모를 향한다' 라고 생각하기 보다, '(MRO 상) 다음 상속받을 대상을 향한다'라고 생각하는 것이 바람직 

- MRO에 따라 각 클래스의 메서드를 찾아가기 때문에, 단순히 직계 부모 뿐만 아니라 시작점에 따라 적절한 상위 클래스를 안전히 호출할 수 있음!

## 에러와 예외

- 문법 에러 : `Invalid Syntax`, `assign to literal`
- 예외 : 프로그램 실행 중 오류
    - 내장 예외


### 예외 처리

- 예외가 발생했을 때 프로그램이 비정상적으로 종료되지 않고, 적절하게 처리할 수 있도록 하는 방법

```python 
try : # 예외가 발생할 수 있는 코드 작성

except : # 예외가 발생했을 때 실행할 코드 작성

else : # 예외가 발생하지 않았을 때

finally # 예외 발생 여부와 상관없이 항상 실행할 코드 작성

```

## 참고 파트

### 예외 처리 주의사항

```python
# 0을 입력했을 때, 출력할 구문은?
# ? 

try:
    num = int(input('100으로 나눌 값을 입력하시오'))

except Exception:
    print('0으로 나눌 수 없습니다.')

except ZeroDivisionError:
    print('0으로 나눌 수 없습니다.')

except:
    print('에러가 발생하였습니다.')


# except Exception이 모든 예외를 가로채기 때문에, 그 아래에 있는 ZeroDivisionError 전용 코드는 절대 실행되지 않음
```

- 구체적인 예외부터 먼저 except 구문에 작성(처리)하고, 이후 코드로 갈 수록 범용적인 Error로 처리하는 것이 바람직 (마지막 : Exception)

## 클래스의 의미와 활용

프로그램 규모가 커질 수록, 변수와 함수만으로 관리하기가 점점 어려워진다. 따라서 클래스 단위로 관련있는 정보와 기능을 '한 덩어리'로 묶어 구조를 명확히 할 수 있음 

추후 알고리즘을 다룰 때에는 보통 활용되지 않지만, '개발'의 관점에서, 어떤 패러다임으로 우리가 코딩을 해야하는 지를 생각해볼 필요가 있다. 

클래스는 컴퓨터 문제 보다는 '현실 세계의 문제'를 해결할 때에 활용되는 도구라 볼 수 있다. (추후 : 백엔드 수업)
