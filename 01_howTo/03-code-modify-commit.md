# 3단계: 코드 수정 및 커밋

브랜치에서 작업하고 변경사항을 커밋합니다.

## 📝 작업 진행

1. 파일 생성/수정
2. 코드 저장
3. 테스트 실행 (필요시)

## 🔧 변경사항 커밋

### 1. 변경된 파일 확인
```bash
git status
```

### 2. 파일을 스테이징 영역에 추가
```bash
# 특정 파일만
git add 파일명

# 모든 파일
git add .
```

### 3. 커밋 메시지와 함께 커밋
```bash
git commit -m "명확한 커밋 메시지"
```

## 💬 커밋 메시지 작성 가이드

```
[타입] 간단한 설명

상세 설명 (필요시)
```

### 타입
#### 여러 종류가 있지만, 저희는 learned를 주로 쓸겁니다.
- `learned:` 새로 배운 내용 
- `Feat:` 새로운 기능
- `Fix:` 버그 수정
- `Docs:` 문서 수정
- `Style:` 코드 스타일 변경
- `Refactor:` 코드 리팩토링

### 예시
```
learned: Python 기본 개념 TIL 추가

- 변수와 자료형 설명
- 코드 예시 포함
- 예제 파일 추가
```

## 📋 커밋 히스토리 확인
```bash
git log --oneline
```
