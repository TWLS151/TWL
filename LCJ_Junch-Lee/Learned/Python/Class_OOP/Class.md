## 객체와 클래스


### 객체(Object)

: **속성**과 **동작**을 가지는 실제 존재하는 사물의 추상화

- 속성 : 변수(데이터)
- 동작 : 메서드

---
### 클래스 (Class)

: 객체를 만들기 위한 설계도 

- 데이터(속성)와 기능(메서드)을 하나의 '묶음'으로 정의하는 설계도(blueprint)


### cf. 클래스 생성자 메서드 (__init__)

- (`_`) : 매직 메서드 (개발자의 의도와 무관하고, 호출 시점을 파이썬이 결정)

- 클래스 생성자 메서드는 '클래스 생성 시 바로' 실행됨

---
### 인스턴스 (Instance)

: 클래스를 통해 생성된 실제 객체 (ex. 자동차 설계도 - 소나타)

각 인스턴스는 클래스 구조에 따라 동작하지만 인스턴스는 서로 독립된 데이터를 가질 수 있음 

---
### 클래스와 인스턴스의 관계

ex. 가수 - 아이유, bts 

- 아이유는 객체다 (O)
- 아이유는 인스턴스다 (X) (클래스 명시 X)
- 아이유는 가수의 인스턴스다 (O) 

- 클래스를 생성한다 == 공통된 특성과 기능을 가진 틀을 만든다

- 이 틀에서 생성된 인스턴스

## **(중요) 사실 우리는 모두 객체 지향적 프로그래밍을 하고 있었다**

#### 데이터들이 메서드를 호출할 수 있었던 이유?

```python
class Singer:
    pass


# iu와 bts는 Singer 클래스의 인스턴스
iu = Singer()
bts = Singer()

print(type(iu)) # <class '__main__.Singer'>
print(type(bts))


# 문자열 변수 name은 (정확히 'Alice') string 클래스의 인스턴스 !
name = "Alice"
print(type(name)) # <class 'str'>

# 리스트 변수 data는 list 클래스의 인스턴스
data = [1, 2, 3]
print(type(data)) # <class 'list'>
```

- 우리가 사용하던 데이터가 곧 클래스의 인스턴스 였기 때문 !!

```python
"hello".upper()

# 문자열.대문자로()
# 객체.행동()
# 인스턴스.메서드()
```
- 행동 중심에서, 객체 중심의 사고로 전환

## 메서드 (Method)

메서드 종류 : `Class`, `Instance`, `Static`


### 인스턴스 메서드

: 인스턴스가 사용하는 메서드 / 인스턴스의 상태 조작, 동작

#### 인스턴스 메서드 구조

- 반드시 첫 번째 인자로 인스턴스 자신을 받음
  - 이름은 `self`로 하고, 바꾸지 않을 것을 강력히 권장

- 인스턴스에 대한 조작을 하기 위해서는 인스턴스 자신을 인자로 가져야 하기 때문

```python
'hello'.upper()

# 실제 파이썬 내부 동작
str.upper('hello')

# str 클래스가 upper 메서드를 호출했고,
# 첫 번째 인자로 문자열 인스턴스가 들어간 것
```
**>>> 인스턴스 메서드의 첫 번째 인자가 반드시 인스턴스 자기 자신인 이유**

- 단축형 호출 : 절차 지향적 메서드를 객체 지향적으로 사용하게끔 변형

---

### 클래스 메서드

: 클래스 변수를 조작하거나, 클래스 레벨의 동작을 수행

ex. 모든 학생이 다니는 '학교 이름'을 설정하는 기능 

#### 구조

- `@classmethod` 데코레이터를 사용하여 정의
- 호출 시, 첫 번째 인자로 반드시 해당 메서드를 호출하는 클래스 `cls` 가 인자로 들어가야 함(강력 권장)

---

### 스태틱(정적) 메서드 

: 클래스, 인스턴스와 상관없이 독립적으로 동작하는 메서드 

- 클래스, 인스턴스 조작과 상관없이 다른 기능을 하는 함수 (일종의 도우미 역할)

- `@staticmethod` 데코레이터를 사용하여 정의
- 자동으로 전달받는 인자는 없음 (ex. `self`, `cls`)

- 개별 객체의 상태를 다룰 땐 반드시 인스턴스 메서드를 사용
