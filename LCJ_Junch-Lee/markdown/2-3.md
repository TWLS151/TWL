## 절차 지향 프로그래밍

: 함수와 로직 중심의 작성, 데이터를 순차적으로 처리

### 특징

- 변수와 함수를 별개로 사용

- 로직에 맞게 함수가 단계 별로 실행이 되는 것 (행위)가 중심

- 변수는 중요치 X 

- 순차적인 명령어 실행이 중요

  
### 한계

- 복잡성 증가에 따른 관리의 어려움

    - 프로그램 규모가 커질 수록 데이터(전역 변수), 함수의 관리가 어려움

- 문제 발생 시 유지/보수가 어려움


## 객체 지향 프로그래밍 

: 순서 보다, 데이터가 중심(주체)이 되어 데이터가 행동을 하는 개념

- 행동하는 주체가 바로 '객체' 

- 클래스는 설계도, 인스턴스는 실제 물건


ex. 자동차 설계도 - 클래스, 조립된 자동차 - 인스턴스


#### 객체 지향 사고 예시

사람 (객체) 안에 name, age와 이와 관련된 기능(메서드) 포함

```python

class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    
    def introduce(self):
        print(f"안녕하세요, 저의 이름은 {name}이고, 나이는 {age}입니다.")

alice = Person("Alice", 25)

alice.introduce()
```


### 특징

- 데이터와 처리하는 함수(메서드)를 하나의 단위(객체)로 묶어서 조직적으로 관리하자 (데이터와 메서드의 결합)

- 처리 순서 보다는, **'객체 간 상호작용'**이 중요

- 데이터의 능동화 : 데이터의 역할이 더욱 커짐 (<-> 절차 지향 : 함수의 재료에 국한)

- 코드의 구조화와 재사용성을 높임 

- 실제 세계의 모델링 방식과 더 유사한 프로그래밍

## 객체와 클래스


### 객체(Object)

: **속성**과 **동작**을 가지는 실제 존재하는 사물의 추상화

- 속성 : 변수(데이터)
- 동작 : 메서드


### 클래스

: 객체를 만들기 위한 설계도 

- 데이터(속성)와 기능(메서드)을 하나의 '묶음'으로 정의하는 설계도(blueprint)


### cf. 클래스 생성자 메서드 (__init__)

- (`_`) : 매직 메서드 (개발자의 의도와 무관하고, 호출 시점을 파이썬이 결정)

- 클래스 생성자 메서드는 '클래스 생성 시 바로' 실행됨

- 이 틀에서 생성된 인스턴스

## **(중요) 사실 우리는 모두 객체 지향적 프로그래밍을 하고 있었다**

#### 데이터들이 메서드를 호출할 수 있었던 이유?

```python
class Singer:
    pass


# iu와 bts는 Singer 클래스의 인스턴스
iu = Singer()
bts = Singer()

print(type(iu)) # <class '__main__.Singer'>
print(type(bts))


# 문자열 변수 name은 (정확히 'Alice') string 클래스의 인스턴스 !
name = "Alice"
print(type(name)) # <class 'str'>

# 리스트 변수 data는 list 클래스의 인스턴스
data = [1, 2, 3]
print(type(data)) # <class 'list'>
```

- 우리가 사용하던 데이터가 곧 클래스의 인스턴스 였기 때문 !!

```python
"hello".upper()

# 문자열.대문자로()
# 객체.행동()
# 인스턴스.메서드()
```
- 행동 중심에서, 객체 중심의 사고로 전환

## 메서드

메서드 종류 : 인스턴스, 클래스, 스태틱 


### 인스턴스 메서드

: 인스턴스가 사용하는 메서드 / 인스턴스의 상태 조작, 동작

#### 인스턴스 메서드 구조

- 반드시 첫 번째 인자로 인스턴스 자신을 받음
  - 이름은 `self`로 하고, 바꾸지 않을 것을 강력히 권장

- 인스턴스에 대한 조작을 하기 위해서는 인스턴스 자신을 인자로 가져야 하기 때문

```python
'hello'.upper()

# 실제 파이썬 내부 동작
str.upper('hello')

# str 클래스가 upper 메서드를 호출했고,
# 첫 번째 인자로 문자열 인스턴스가 들어간 것
```
**>>> 인스턴스 메서드의 첫 번째 인자가 반드시 인스턴스 자기 자신인 이유**

- 단축형 호출 : 절차 지향적 메서드를 객체 지향적으로 사용하게끔 변형

### 클래스 메서드

: 클래스 변수를 조작하거나, 클래스 레벨의 동작을 수행

ex. 모든 학생이 다니는 '학교 이름'을 설정하는 기능 

#### 구조

- `@classmethod` 데코레이터를 사용하여 정의
- 호출 시, 첫 번째 인자로 반드시 해당 메서드를 호출하는 클래스 `cls` 가 인자로 들어가야 함(강력 권장)


### 스태틱(정적) 메서드 

: 클래스, 인스턴스와 상관없이 독립적으로 동작하는 메서드 

- 클래스, 인스턴스 조작과 상관없이 다른 기능을 하는 함수 (일종의 도우미 역할)

- `@staticmethod` 데코레이터를 사용하여 정의
- 자동으로 전달받는 인자는 없음 (ex. `self`, `cls`)

- 개별 객체의 상태를 다룰 땐 반드시 인스턴스 메서드를 사용 

### 결국 '무엇을 바꾸는가?'가 메서드 선택의 기준


#### 사실 클래스, 인스턴스 모두 클래스 / 인스턴스 / 스태틱 메서드를 모두 호출할 수 있음
