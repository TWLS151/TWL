# Git Multiple PR Workflow
2026-01-24

## situation(상황)
깃을 쓸 때 브랜치를 만들어서 푸쉬이후에 메인 브랜치에 PR을 했다. 그런데 PR 승인이 나기 이전에 추가 공부를 해서 두번째 새 브렌치를 만들어서 깃에 푸쉬를 했다.
그러면 두번째 브렌치도 main에 PR을 하면 뭔가 main에서 꼬일것 같다라는 생각을 했다.
## task(목표)
두개의 브랜치를 main에 에러 없이 합치기
## Action(나의 행동)
AI검색결과, chained PR(체인 PR)을 쓰면 좋다라고 한다. 
## Result(결과)
지금 생각해보니 다른 사람들이 같이 PR을 요청한 상황과 같은 것 같다. 따라서 rebase를 통해서 conflict를 해결하면 될 것 같다.
차이점이 있다면 지금은 새로 만든 두 브랜치가 모두 내가 만든것이기 때문에 체인 PR(chained PR)이라는 방법도 쓸 수 있다.
체인 PR은 첫번째 만든 브랜치에 두번째 만든 브랜치를 합치면 된다. 두 브랜치가 모두 내 내용이어서 merge해도 된다.


### 추가 사항
-  동일 브랜치 수정 시
• 기존 PR 브랜치에서 추가 커밋 후 푸시하면 기존 PR에 자동으로 반영됨.
• 이전 PR을 삭제할 필요 없음.
-  별도 브랜치 작업 시
• 독립적 작업: `main`에서 새 브랜치를 생성했다면 `main`을 대상으로 새 PR 생성.
• 의존적 작업 (Chained PR): 이전 브랜치에서 새 브랜치를 생성했다면, `base`를 이전 브랜치로 설정하여 PR 생성.
